最近我以合法手段要到了学校一台服务器的使用权限，就打算尝试一下绕过校园网计费。于是研究了一下

理解代理

作为中国的程序员，“代理”这两个字应该是十分熟悉的。所谓代理，就是让自己的网络流量通过服务器流向网站。但之前我一直没有详细了解过它们的实现，趁这个机会学习了一下。

代理有多种HTTP，SSL，Socks5，FTP等等。但常用的一般是HTTP和Socks5两种代理。为了达到我使用校园网的目的，我优先尝试了Socks5代理。

Socks5

Socks5代理因为是直接建立双方连接，所以不关心流量走的是什么协议。无论是HTTPS还是HTTP还是WSS都可以走Socks5代理，RFC的文档在标题，另附上Wiki的说明。

然后按照RFC文档我实现了一个Socks5代理服务器如下:

import logging
import socket
import selectors
import struct
import threading
import time
import re
from multiprocessing import Process
import traceback

logging.basicConfig(
    level=logging.DEBUG,
    format='[%(levelname)s]-[%(asctime)s]-[%(lineno)s]==> %(message)s',
    datefmt='%Y-%m-%d %H:%M:%S',
)
logger = logging.getLogger(__file__)


class Socks5:
    """Socks5代理"""

    def __init__(self, IP: str = "0.0.0.0", port: int = 1080):
        self.IP = IP
        self.port = port
        self.server = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
        self.server.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)
        self.server.bind((IP, port))
        self.server.listen(5)
        logger.info("Starting Server.")

    def __del__(self):
        self.server.close()
        logger.info('Stopping Server.')

    def bridge(self, local, remote):
        sel = selectors.SelectSelector()
        sel.register(local, selectors.EVENT_READ)
        sel.register(remote, selectors.EVENT_READ)
        die = False
        while not die:
            try:
                event_list = sel.select()
                for key, events in event_list:
                    conn = key.fileobj
                    if conn == local:
                        temp = local.recv(4096)
                        if not temp:
                            die = True
                            break
                        if remote.send(temp) <= 0:
                            break
                    if conn == remote:
                        temp = remote.recv(4096)
                        if not temp:
                            die = True
                            break
                        if local.send(temp) <= 0:
                            break
            except:
                traceback.print_exc()
                break

    def __link(self, sock):
        sock.recv(256)
        sock.send(b"\x05\x00")
        data = sock.recv(4) or '\x00' * 4
        CMD = data[1]
        if CMD != 1:
            return
        ATYP = data[3]
        if ATYP == 1:
            ipv4 = sock.recv(4)
            remote_addr = socket.inet_ntoa(ipv4)
        elif ATYP == 3:
            addr_len = int.from_bytes(sock.recv(1), byteorder='big')
            remote_addr = sock.recv(addr_len).decode()
        elif ATYP == 4:
            ipv6 = sock.recv(16)
            remote_addr = socket.inet_ntop(socket.AF_INET6, ipv6)
        else:
            return
        # DST.PORT
        remote_addr_port = struct.unpack('>H', sock.recv(2))[0]

        # 返回给客户端 success
        reply = b"\x05\x00\x00\x01"
        reply += socket.inet_aton("0.0.0.0") + struct.pack(">H", self.port)
        sock.send(reply)

        # 拿到 remote address 的信息后，建立连接
        logger.debug(f"Connecting {remote_addr}:{remote_addr_port}")
        try:
            remote = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
            remote.connect((remote_addr, remote_addr_port))
        except socket.error as e:
            logger.error(f"{remote_addr}:{remote_addr_port} ==> {e}")
            sock.close()
            return

        self.bridge(sock, remote)

    def _worker(self):
        while True:
            try:
                sock, address = self.server.accept()
                client = threading.Thread(target=self.__link, args=(sock,), daemon=True)
                client.start()
            except socket.error as error:
                logger.error(error)

    def run(self):
        worker = Process(target=self._worker)
        worker.daemon = True
        worker.start()
        try:
            while True:
                time.sleep(1)
        except KeyboardInterrupt:
            pass


if __name__ == '__main__':
    S = Socks5()
    S.run()

socks
这个第三方库支持三种代理 Socks4, Socks5和HTTP。安装方式是pip install pysocks

HTTP代理

通过谷歌得知，HTTP代理分两种。

  • 在HTTPS技术还没出来的时候，人们代理网站，只需要把请求的URI带上完整的网址就行了。代理服务器会代替你访问这个网站，然后把结果回馈给你。
    中间人攻击就是这种操作。

  • 在HTTPS普及之后，这种操作已经不行了，因为证书对不上。所以有了新的操作：CONNECT 方法。
    这种方法的实现细节是 用户向代理服务器发送CONNECT (domain|IPV4|IPV6):Port (HTTP/1.1 | HTTP/1.0 | HTTP/2.0)的数据包，代理服务器视情况返回一个状态。当客户端接收到200状态的时候就意味着代理服务器与目标服务器建立了TCP连接，可以直接发送普通的请求过去。此时数据包会由一个TCP连接发送到代理服务器，代理服务器原封不动的将数据包转发给目标服务器。
    这种方法跟Socks5代理十分相似，代理服务器不关心你们使用的是什么交互方式，它只需要把两个TCP连接拼在一起就行了。

那么，我们上面提到的不支持的HTTP代理就是第一种代理了。看了看socks的源代码，嗯，只支持第二种代理。

但学校这台代理服务器的逻辑是什么？我想搞清楚。发送了一个错误的包过去，果然从错误页面拿到了这台代理服务器的信息—— squid2.6。一番谷歌，squid的默认配置就是只允许向443端口使用CONNECT方法。

题外话

既然提到了代理，那顺便提一下。现在的Shadowsocks(R)都是基于Socks5代理的。只是他们对传输的数据做了对称加密/混淆，从而使审查机制无法得知他们想要访问的是什么。

如果我们选择不开源，自己使用Socks5代理配合自己写的混淆方法来对抗审查，是完全可以做到的。经我一友人实验，效果很不错。

ShadowsocksR更有意思的是，故意加入某些网站的头，从而使得审查机制认为我们只是使用Socks代理访问白名单中的网站。有人发现还能用这个方法来绕过运营商的流量计费😄
