浅谈设备自适应

/编程杂记/2018-02-20/

准备工作

对于不同设备，不同浏览器，它们对于页面的默认实现效果不一定一样。所以我们需要在编写自适应页面之前，做一些准备工作。

CSS

在以前，IE一家独大，后来微软作死把自己作死了。现在大多是以Chrome/Firefox这两家的为准。然而就算是这两家的浏览器，在对页面渲染的默认行为上也有着不同。

所以重置CSS效果是十分重要的一件事情。一般来说，大型CSS框架会自定义几乎所有的HTML标签，所以可以不需要Reset.css。但如果我们不使用一个大而全的框架，那么在编写CSS的过程中就要十分小心浏览器的默认行为。

对此，我推荐在不想使用Css框架的情况下，使用Normalize.css。

Normalize.css是一个很小的CSS文件，但它在默认的HTML元素样式上提供了跨浏览器的高度一致性。现在已经被用于Twitter Bootstrap、HTML5 Boilerplate、Rdio、CSS Tricks 以及许许多多其他框架、工具和网站上。
它的Github地址 https://github.com/necolas/normalize.css

字体

传统上，网页都是调用本地的字体库来显示页面字体，但是现在设备那么多种，虽然大佬们研究出了一套安全字体(1)的列表，但事实上，效果并不好。

此时谷歌推出的在线字体库就显得十分有用了。对于繁体汉字，在我写下这篇文章的时候，已经有五种字体可供使用，其中我最喜欢的一种可以在我的about看到。
而简体汉字只有仅有的两种

@import url(https://fonts.googleapis.com/earlyaccess/notosansscsliced.css);
@import url(https://fonts.googleapis.com/earlyaccess/notosanssc.css);

分别对应着

font-family: 'Noto Sans SC Sliced', sans-serif;
font-family: 'Noto Sans SC', sans-serif;

至于拉丁世界的字体那就太多了，毕竟也就几十个字符的事情。

插一句，字体大小也是十分重要的，我的建议是在body中使用px单位设定好一个基础字体大小，然后其他标签里都使用em，这样在下面使用媒体查询时，只需要修改body的字体大小就可以轻松对不同大小设备做字体大小适配。

编写

自适应页面大体可以分为两种，一种是查询屏幕的高宽来确定该使用哪一些CSS进行渲染，雅虎的Pure，推特的Bootstrap都是这样做的。另一种就是flex布局，让浏览器来决定你页面的渲染效果。

其实说是两种，它们却并不冲突，可以混合使用。恰当的混合使用效果会十分出色。

媒体查询

例如下例，它的含义就是在屏幕宽度不小于1300px时使用其中的CSS。

<style>
@media (min-width: 1300px) {
    article{
        margin: auto;
    }
}
</style>

或者@media (max-width: 900px) {}，来设定屏幕宽度不大于900px的行为。

你也可以在link中使用，来决定使用哪一套css。

<link rel="stylesheet" media="(max-width: 800px)" href="example.css" />

媒体查询还有逻辑运算等等，可以在MDN上看到更多使用方法。

Flex布局

首先让我们看看MDN上对此的描述

CSS3 弹性盒子(Flexible Box 或 Flexbox)，是一种用于在页面上布置元素的布局模式，使得当页面布局必须适应不同的屏幕尺寸和不同的显示设备时，元素可预测地运行。对于许多应用程序，弹性盒子模型提供了对块模型的改进，因为它不使用浮动，flex容器的边缘也不会与其内容的边缘折叠。
在定义方面来说，弹性布局是指通过调整其内元素的宽高，从而在任何显示设备上实现对可用显示空间最佳填充的能力。弹性容器扩展其内元素来填充可用空间，或将其收缩来避免溢出。
块级布局更侧重于垂直方向、行内布局更侧重于水平方向，与此相对的，弹性盒子布局算法是方向无关的。虽然块级布局对于单独一个页面来说是行之有效的，但其仍缺乏足够的定义来支持那些必须随用户代理(user agent)不同或设备方向从水平转为垂直等各种变化而变换方向、调整大小、拉伸、收缩的应用程序组件。 弹性盒子布局主要适用于应用程序的组件及小规模的布局，而（新兴的）栅格布局则针对大规模的布局。这二者都是 CSS 工作组为在不同用户代理、不同书写模式和其他灵活性要求下的网页应用程序有更好的互操作性而做出的更广泛的努力的一部分。

Flex布局以轴为主线，分主轴和侧轴，有时候也会叫竖轴和横轴(横竖嘛🙂)。

通过定义display: flex或者display: inline-flex来使用。

可以查看MDN和MDN来获取详细语法。

(1): 所有平台都预装的字体，被称为"安全字体"，因为它可以保证所有用户的视觉效果是一样的。
