简介

从Wiki上我们可以看到这样一段描述

动态规划（英语：Dynamic programming，简称DP）是一种在数学、管理科学、计算机科学、经济学和生物信息学中使用的，通过把原问题分解为相对简单的子问题的方式求解复杂问题的方法。

乍一看，这描述和分治法那么像，都是分割原问题为简单问题，“大事化小，小事化了”。但事实上，这真的不一样。且听我慢慢道来。

浅谈

在大一的离散数学课本上我见过图的单源最短路径算法——Dijkstra算法。

它讲述了一连通图中，从起点出发，找到到达目标点的最短路径的方法。

这样一张图，找到从abcd到^的最短路径。如果穷举结果，一个正常人类很难短时间内遍历完它们。

然而Dj算法从是这样做的

从abcd开始，能直接到达四个节点，分别记录下这四个节点和abcd的距离。

然后从这四个节点分别出发，能直接到达的节点有六个。分别记录下这六个节点与abcd的最短的距离。(因为从abcd到这六个节点都有多种路径，而在此我们只找最短路径，所以只记录最短路径距离，最长路径同理。)

重复上一步，直到到达 ^。

能看出，这就是一个图的广度优先遍历而已。

而这正是DP！

从起始状态abcd到状态^，中间一共要经过四次状态变化。

  1. 每个中间状态都拥有一条符合我们要求的路径——最短的路径 (最优子结构)。
  2. 每一层的中间状态确定之后，它们到abcd的最短路径不会因为后层而改变 (无后效性)。
  3. 类似于ab的中间状态，有多个下层中间状态使用了它的最短路径 (子问题重叠)。

实例

吹气球问题

题目描述

有n个气球，编号为0到n-1，每个气球都有一个分数，存在 nums数组中。每次吹气球i可以得到的分数为nums[left] * nums[i] * nums[right]，left 和 right 分别表示i气球相邻的两个气球。当i气球被吹爆后，其左右两气球即为相邻。要求吹爆所有气球，得到最多的分数。

  • 补注:

    (1) 假设 nums[-1] = nums[n] = 1

    (2) 0 ≤ n ≤ 500, 0 ≤ nums[i] ≤ 100

  • 样例:

    输入: 3, 1, 5, 8

    输出: 167
